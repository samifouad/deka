# deka start

Start the full Deka hosting stack.

```bash
deka start
```

## Options

- `--genesis` - Initialize genesis block on first start
- `-d, --detach` - Run in background (default: true)
- `--infra` - Start only infrastructure services (postgres, redis) and expose ports for local dev

Infrastructure mode exposes:

- postgres: `postgres://tana:tana_dev_password@localhost:5432/tana`
- redis: `redis://localhost:6380`
- pgbouncer: `postgres://tana:tana_contract_password@localhost:6432/<db>`

## Example output
