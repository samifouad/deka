PHPX supports JSX with typed props and server rendering.
You can keep rendering simple (SSR only) or add hydration when needed.

## Basic component

```phpx
interface CardProps {
  $title: string
  $subtitle: Option<string>
}

function Card({ $title, $subtitle }: CardProps) {
  return <div class="card">
    <h2>{$title}</h2>
    {$subtitle ? <p class="muted">{$subtitle}</p> : null}
  </div>
}
```

## Render in a page

```phpx
---
import { renderToString } from "component/dom"
import { Card } from "@/components/Card"
---
<!doctype html>
<html>
  <body>
    <div id="app">{renderToString(<Card title="Hello PHPX" subtitle="Server rendered" />)}</div>
  </body>
</html>
```

Expected output HTML (simplified):

```html
<div id="app"><div class="card"><h2>Hello PHPX</h2><p class="muted">Server rendered</p></div></div>
```

## Invalid props example

```phpx
<Card title="Hello" subtitle={123} />
```

Expected behavior:

```text
Type error: subtitle expects Option<string>.
```

## Notes

- Props are passed as one object and can be destructured in the parameter list.
- JSX is PHPX-only syntax.
- Hydration behavior is explicit; static output works without client runtime.
