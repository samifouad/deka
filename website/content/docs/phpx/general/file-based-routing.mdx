PHPX supports file-based routing so apps do not need a hand-written router for common cases.

## Route roots

- `app/` for page routes
- `api/` for endpoint routes

## `app/` conventions

- `page.phpx` defines the route leaf
- `layout.phpx` defines shared layout for that segment
- dynamic segments use `[param]`
- wildcard segments use `[...rest]`

Example tree:

```text
app/
  layout.phpx
  page.phpx
  users/
    layout.phpx
    page.phpx
    [id]/
      page.phpx
  docs/
    [...slug]/
      page.phpx
```

Examples:

- `app/page.phpx` -> `/`
- `app/users/page.phpx` -> `/users`
- `app/users/[id]/page.phpx` -> `/users/:id`
- `app/docs/[...slug]/page.phpx` -> `/docs/*`

## `api/` conventions

HTTP method files map per route:

- `get.phpx`
- `post.phpx`
- `put.phpx`
- `patch.phpx`
- `delete.phpx`

Example tree:

```text
api/
  packages/
    get.phpx
    post.phpx
  packages/
    [name]/
      get.phpx
      delete.phpx
```

Examples:

- `api/packages/get.phpx` -> `GET /api/packages`
- `api/packages/post.phpx` -> `POST /api/packages`
- `api/packages/[name]/delete.phpx` -> `DELETE /api/packages/:name`

## Module imports with file routing

Routing and module organization are separate concerns.
You can still import components by alias:

```phpx
import { Card } from "@/components/Card"
```

`app/` controls URL structure. `components/` controls UI/module organization.
