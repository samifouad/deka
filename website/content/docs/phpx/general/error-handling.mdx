PHPX favors explicit result values over hidden throw paths.

## Result flow

```phpx
function parseCount($raw: string): Result<int, string> {
  if (!isNumeric($raw)) {
    return Err("not numeric")
  }
  return Ok(toInt($raw))
}

$count = parseCount("7")
if (isErr($count)) {
  echo "bad-input"
  return
}
echo unwrap($count)
```

Expected output:

```text
7
```

## Option flow

```phpx
function usernameOrGuest($name: Option<string>): string {
  return match ($name) {
    Some($value) => $value
    None => "guest"
  }
}
```

## Guidance

- return `Result<T, E>` when operations can fail
- use `Option<T>` when value may not exist
- keep error types narrow and descriptive
